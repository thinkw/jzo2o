# 开发日志 2026-05-17

## Chat 组件 LaTeX 公式渲染修复

### 问题背景

AI 助手返回的数学公式使用 `[...]` 包裹（如 `[\frac{a}{b}]`），但 Cherry Markdown 只识别标准的 `$...$` 和 `$$...$$` LaTeX 格式，导致数学公式无法渲染。

### 方案对比

| 方案 | 改动量 | 误判风险 | 维护成本 | 评价 |
|------|--------|----------|----------|------|
| 前端正则预处理 | ~50行 | 有（多层防护复杂） | 高 | ❌ 治标 |
| 后端 System Prompt | **5行** | **无** | **零** | ✅ 治本 |

### 最终方案：AI Engine 注入 System Prompt

修改文件：`jzo2o-ai-engine/app/api/chat.py`

**核心改动：**

1. 新增 `SYSTEM_PROMPT` 常量，要求 LLM 使用标准 Markdown LaTeX 格式：
   - 行内公式用 `$...$`
   - 块级公式用 `$$...$$`
   - 禁止使用 `[...]` 作为公式分隔符

2. 在请求处理时，将 system prompt 插入消息队列头部：
   ```python
   messages_dict.insert(0, {"role": "system", "content": SYSTEM_PROMPT})
   ```

### 效果

从源头强制 LLM 输出标准格式，彻底解决渲染问题，无需前端做任何特殊处理。
